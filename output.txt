STREAM TYPE	PURPOSE	REAL-WORLD EXAMPLE
READABLE	READ DATA FROM SOURCE	FS.CREATEREADSTREAM('FILE.TXT')
WRITABLE	WRITE DATA TO DESTINATION	FS.CREATEWRITESTREAM('OUTPUT.TXT')

2. 📦 ROLE OF BUFFER CLASS
WHAT THEY'LL ASK:
"WHY DO WE NEED BUFFERS IN NODE.JS?"

"WHAT'S THE DIFFERENCE BETWEEN BUFFER AND STRING?"

"HOW DO YOU CREATE A BUFFER?"

YOUR ANSWER:
BUFFER = TEMPORARY STORAGE FOR BINARY DATA (RAW MEMORY ALLOCATION OUTSIDE V8 HEAP).
NODE.JS USES BUFFERS BECAUSE:

JAVASCRIPT STRINGS ARE UTF-8 ENCODED

FILES, NETWORKS, STREAMS SEND RAW BINARY (TCP, HTTP, FILESYSTEM)

CODE EXAMPLES:
JS
// CREATE BUFFER
CONST BUF1 = BUFFER.FROM('HELLO', 'UTF-8');  // <BUFFER 48 65 6C 6C 6F>
CONST BUF2 = BUFFER.ALLOC(10);               // EMPTY BUFFER OF 10 BYTES
CONST BUF3 = BUFFER.ALLOCUNSAFE(10);        // FASTER BUT UNSAFE (MAY CONTAIN OLD DATA)

// CONVERT BUFFER TO STRING
BUF1.TOSTRING();  // "HELLO"

// BUFFER OPERATIONS
BUF1.LENGTH;      // 5 (BYTES)
BUF1[0];          // 72 (ASCII FOR 'H')
KEY POINT: WHEN YOU DO FS.READFILE() WITHOUT ENCODING, YOU GET A BUFFER. WHEN YOU PIPE STREAMS, BUFFERS ARE AUTOMATICALLY HANDLED.

3. ⚡ EVENT-DRIVEN PROGRAMMING
WHAT THEY'LL ASK:
"WHAT DOES 'EVENT-DRIVEN' MEAN IN NODE.JS?"

"HOW IS IT DIFFERENT FROM TRADITIONAL REQUEST-RESPONSE?"

"GIVE ME AN EXAMPLE OF EVENT-DRIVEN CODE."

YOUR ANSWER:
EVENT-DRIVEN = FLOW CONTROLLED BY EVENTS (CLICKS, MESSAGES, COMPLETED TASKS) NOT SEQUENTIAL EXECUTION.

NODE.JS HAS:

EVENT LOOP (WAITS FOR EVENTS)

EVENT EMITTERS (TRIGGER EVENTS)

LISTENERS (REACT TO EVENTS)

TRADITIONAL VS EVENT-DRIVEN:
TRADITIONAL (BLOCKING)	EVENT-DRIVEN (NON-BLOCKING)
READ FILE → WAIT → PRINT	READ FILE → PROVIDE CALLBACK → CONTINUE
ONE THING AT A TIME	MULTIPLE THINGS WITH LISTENERS
EXAMPLE:
JS
// REAL EVENT-DRIVEN: HTTP SERVER
CONST SERVER = HTTP.CREATESERVER((REQ, RES) => {
  RES.END('HELLO');
}); // WAITS FOR 'REQUEST' EVENT

// CUSTOM EVENT EMITTER
CONST EVENTEMITTER = REQUIRE('EVENTS');
CONST MYEMITTER = NEW EVENTEMITTER();

MYEMITTER.ON('USERLOGGEDIN', (NAME) => {
  CONSOLE.LOG(`${NAME} JUST LOGGED IN`);
  // SEND EMAIL, UPDATE DATABASE, ETC.
});

MYEMITTER.EMIT('USERLOGGEDIN', 'ANNUVRAT'); // TRIGGERS THE EVENT
KEY POINT: MAKES NODE.JS GREAT FOR I/O-HEAVY, REAL-TIME APPS (CHAT, GAMING, STREAMING).

4. 📡 EVENTS MODULE
WHAT THEY'LL ASK:
"WHAT DOES THE 'EVENTS' MODULE PROVIDE?"

"HOW DO YOU CREATE A CUSTOM EVENT EMITTER?"

"WHAT'S THE DIFFERENCE BETWEEN ON() AND ONCE()?"

YOUR ANSWER:
THE EVENTS MODULE PROVIDES THE EVENTEMITTER CLASS — THE BACKBONE OF NODE.JS ASYNC ARCHITECTURE.

KEY METHODS:
METHOD	WHAT IT DOES
ON(EVENT, LISTENER)	ATTACH LISTENER (RUNS EVERY TIME EVENT FIRES)
ONCE(EVENT, LISTENER)	RUNS ONCE, THEN AUTO-REMOVED
EMIT(EVENT, ARGS...)	TRIGGER AN EVENT
REMOVELISTENER(EVENT, LISTENER)	REMOVE SPECIFIC LISTENER
REMOVEALLLISTENERS([EVENT])	REMOVE ALL LISTENERS
CODE EXAMPLE:
JS
CONST EVENTEMITTER = REQUIRE('EVENTS');
CONST ORDEREMITTER = NEW EVENTEMITTER();

// LISTENERS
ORDEREMITTER.ON('ORDERPLACED', (ORDERID) => {
  CONSOLE.LOG(`PROCESSING ORDER ${ORDERID}`);
});

ORDEREMITTER.ONCE('CONNECT', () => {
  CONSOLE.LOG('CONNECTED TO DB'); // RUNS ONLY FIRST TIME
});

ORDEREMITTER.ON('ERROR', (ERR) => {
  CONSOLE.ERROR('SOMETHING BROKE:', ERR);
});

// EMIT EVENTS
ORDEREMITTER.EMIT('CONNECT');
ORDEREMITTER.EMIT('ORDERPLACED', 'ORD-123');
ORDEREMITTER.EMIT('ORDERPLACED', 'ORD-456');
ORDEREMITTER.EMIT('CONNECT'); // WON'T RUN (ONCE)
ORDEREMITTER.EMIT('ERROR', NEW ERROR('TIMEOUT'));
🔥 COMMON INTERVIEW TRAP:
"WHAT HAPPENS IF YOU DON'T HANDLE 'ERROR' EVENTS?"
→ NODE.JS WILL THROW AND CRASH THE PROCESS.

MAX LISTENERS WARNING: BY DEFAULT, 10 LISTENERS PER EVENT. USE EMITTER.SETMAXLISTENERS(20) TO INCREASE.
